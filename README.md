Оно **не заменяет** облачные API (OpenAI/Claude) для продакшена, не гарантирует 100% точность ответов и зависит от ресурсов вашего железа. Автоматизированный набор скриптов и конфигураций для развёртывания приватного ИИ-ассистента на базе **Ollama + Open WebUI** с поддержкой RAG, GPU-ускорением и локальным хранением данных.

---

## 🚀 Быстрый старт

### 1️⃣ Требования
- Windows 11 Pro 22H2+ (с поддержкой WSL2)
- NVIDIA GPU с ≥6 ГБ VRAM (RTX 3070 рекомендуется)
- 32 ГБ ОЗУ, 100+ ГБ свободного места на NVMe SSD
- PowerShell 7+ (от администратора) + WSL2 с Ubuntu

### 2️⃣ Установка (строго в указанном порядке)
```powershell
# 1. Клонируйте репозиторий и перейдите в папку
git clone https://github.com/thrxx/ollama-stack.git
cd ollama-stack

# 2. Запустите полную установку (от администратора)
.\install-all.ps1

# 3. Перезагрузите компьютер

# 4. Запустите ассистента
.\start-assistant.ps1
```

✅ **Готово!** Откройте браузер: http://localhost:3000

## 🔐 Аутентификация: 3 независимых контекста

| Контекст | Где хранится | Для чего используется |
|----------|--------------|------------------------|
| **Windows User** | SAM / Windows Hello | Доступ к файлам, запуск скриптов |
| **WSL/Linux User** | `/etc/passwd` | Права на файлы в WSL, выполнение `ollama` |
| **WebUI User** | SQLite (`webui.db`) | **Только** для входа в веб-интерфейс |

⛔ **Важно:** Пароли не синхронизируются. Учётная запись WebUI создаётся **только** при первом входе в `http://localhost:3000`.

## 🌐 Сеть и доступ из локальной сети

- **Localhost:** Работает «из коробки» (`http://localhost:3000`, `http://localhost:11434`)
- **LAN:** Требуется вручную открыть порты в брандмауэре **только для профиля `Private`**
- **API:** Ollama по умолчанию слушает `127.0.0.1:11434`. Для доступа из сети:
  ```powershell
  # В PowerShell (администратор):
  $env:OLLAMA_HOST = "0.0.0.0"
  ollama serve
  ```

⚠️ **Безопасность:** Не открывайте порт `11434` в публичных сетях без аутентификации и HTTPS.

📖 Подробно: [Настройка сети и брандмауэра](docs/networking.md)

## 💾 Хранение данных и бэкапы

⛔ **Никогда не храните модели в `/mnt/c/`** — файловая система 9P снижает скорость загрузки и может нарушить целостность файлов. Используйте нативный путь WSL: `~/.ollama/`.

| Метод | Частота | Надёжность | Что бэкапит |
|-------|---------|------------|-------------|
| `.\backup.ps1` (ZIP) | Еженедельно | ⭐⭐⭐⭐ | Модели + чаты + конфиги |
| `restic` (шифрование) | 2–3 раза/нед | ⭐⭐⭐⭐⭐ | Инкрементально, с дедупликацией |
| Снапшот `.vhdx` | Перед обновлениями | ⭐⭐⭐ | Весь диск WSL (тяжёлый) |
| `rclone` → Cloud | Еженедельно | ⭐⭐⭐ | Только чаты и конфиги (без моделей) |

📖 Подробно: [Стратегии бэкапа и восстановление](docs/backup.md)

## 🚦 Устранение неполадок

| Симптом | Решение |
|---------|---------|
| `CUDA out of memory` | Используйте квантование `q4_k_m`, уменьшите `num_ctx` до 4096 |
| `Connection refused` на 11434 | Проверьте, запущен ли Ollama: `ollama ps`; перезапустите: `Restart-Service Ollama` |
| WebUI не видит Ollama | Убедитесь, что `OLLAMA_BASE_URL=http://host.docker.internal:11434` в `docker-compose.yml` |
| Медленная генерация (<10 т/с) | Проверьте `ollama ps` — модель не должна быть в RAM; используйте `num_gpu: 99` в `config.json` |
| Docker не видит GPU | В Docker Desktop: Settings → WSL Integration → ✅ Enable; `wsl --update` |
| Порт 3000 занят | Найдите процесс: `netstat -ano \| findstr :3000`; измените порт в `docker-compose.yml` |

Полный список: [docs/troubleshooting.md](docs/troubleshooting.md)
