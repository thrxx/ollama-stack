## [1.0.0] - 2026-04-24

### Добавлено
- **Автоматическая установка** (`install-all.ps1`):
  - Проверка прав администратора, виртуализации BIOS, наличия NVIDIA GPU и свободного места на диске.
  - Автоматическая установка WSL2 + Ubuntu 22.04.
  - Скачивание и установка Ollama для Windows.
  - Скачивание и установка Docker Desktop.
  - Генерация `docker-compose.yml` с оптимизированным конфигом Open WebUI (RAG, аналитика отключена, лимиты ресурсов).
  - Создание `config.json` для Ollama с параметрами `num_gpu: 99`, `num_batch: 512`, `num_ctx: 8192`.
- **Быстрый запуск** (`start-assistant.ps1`):
  - Проверка и автозапуск службы Windows `Ollama`.
  - Валидация доступности Ollama API (`127.0.0.1:11434`).
  - Проверка состояния Docker и `docker-compose.yml`.
  - Запуск контейнера Open WebUI с ожиданием готовности (до 10 попыток).
  - Автоматическое открытие браузера по `http://localhost:3000`.
  - Вывод списка загруженных моделей и подсказок по мониторингу.
- **Система бэкапов** (`backup.ps1`):
  - Создание снапшота с меткой времени `yyyy-MM-dd_HHmmss`.
  - Копирование `%USERPROFILE%\.ollama\` и `./webui-data/` через `robocopy /MT:8`.
  - Расчёт размера бэкапа до и после копирования.
  - Автоматическая ротация: удаление папок старше 7 дней.
- **Диагностика** (`check-ai.ps1`):
  - Пошаговая проверка: Ollama (служба + API), Docker, Open WebUI (контейнер + HTTP), NVIDIA GPU.
  - Подсчёт статусов: `OK`, `WARN`, `FAIL` с цветовым кодированием.
  - Режим `-Detailed`: вывод режима WSL2/Hyper-V, потребления RAM/VRAM, свободного места на диске.
  - Fallback-парсинг `docker compose ps --format json` при ошибках десериализации.

### Исправлено
- Скрипты используют единый набор функций вывода (`Write-Step`, `Write-OK`, `Write-Fail`, `Write-Warn`, `Write-Info`) для консистентности CLI.
- `docker-compose.yml` генерируется с `restart: unless-stopped` и ограничением памяти контейнера до `4G`.
- Бэкапы теперь используют `/R:1 /W:1` в `robocopy`, чтобы не блокировать выполнение при временных ошибках файловых систем.
- Обработка кодов выхода `robocopy` (0–7 считаются успешными).
- Корректное определение статуса контейнеров Open WebUI при изменении формата вывода `docker compose ps`.
- Защита от падения диагностики при отсутствии установленных моделей (`ollama list` возвращает только заголовок).
