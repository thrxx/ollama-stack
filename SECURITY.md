## 🛡️ Архитектура безопасности и известные ограничения

Проект предназначен для использования в **доверенной локальной среде**. Ниже приведены границы доверия и технические ограничения каждого слоя:

| Компонент | Модель безопасности | Известные ограничения |
|-----------|---------------------|------------------------|
| **Ollama API** | По умолчанию привязан к `127.0.0.1:11434`. Встроенная аутентификация отсутствует. | Установка `OLLAMA_HOST=0.0.0.0` открывает API без пароля. Не предназначен для прямого доступа из интернета или ненадёжных LAN-сетей. |
| **Open WebUI** | Запускается в изолированном Docker-контейнере. Опциональная аутентификация настраивается при первом входе. | По умолчанию включена для удобства, но требует ручной настройки пароля. Сессии хранятся в локальной SQLite БД (`webui-data/`). |
| **Docker + WSL2** | Виртуализированная сеть, стандартные профили безопасности Docker. Контейнеры работают без привилегий `root` по умолчанию. | Доступ к Docker Socket (`/var/run/docker.sock`) эквивалентен правам администратора хоста. Никогда не монтируйте сокет внутрь пользовательских контейнеров. |
| **PowerShell-скрипты** | `install-all.ps1` требует прав администратора. Остальные скрипты работают от текущего пользователя. Скачивание идёт только с официальных доменов. | Политика выполнения (`ExecutionPolicy`) должна разрешать запуск. Всегда проверяйте контрольные суммы или подписи при скачивании установщиков вручную. |
| **Хранение данных** | Все данные остаются на локальном диске: `%USERPROFILE%\.ollama\`, `./webui-data/`. Телеметрия отключена (`DISABLE_ANALYTICS=true`). | Диск WSL2 (`ext4.vhdx`) и тома Docker не шифруются по умолчанию. Для защиты от физической кражи используйте BitLocker или VeraCrypt. |

## ✅ Рекомендации для пользователей

Чтобы поддерживать высокий уровень безопасности в локальной среде:

- 🔐 **Немедленно включите аутентификацию** в Open WebUI после первого входа (`Настройки` → `Безопасность`).
- 🚫 **Никогда не пробрасывайте порт `11434`** на роутер или в публичную сеть.
- 🌐 При необходимости доступа из LAN используйте **Reverse Proxy** (Nginx/Caddy) с HTTPS и базовой аутентификацией.
- 🔄 **Регулярно обновляйте компоненты**: Windows, Docker Desktop, Ollama, образы Open WebUI.
- 💾 **Шифруйте бэкапы** перед выгрузкой во внешние хранилища (`restic`, `7-Zip` с паролем, `rclone crypt`).
- 🧹 **Удаляйте неиспользуемые модели**: `ollama rm <модель>` снижает поверхность атаки и экономит место.
- 🛑 **Запускайте скрипты только из этого репозитория**. Проверяйте URL при `git clone`.
